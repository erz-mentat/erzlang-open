## CLI replay lane

Checked-in fixture packs can now be replayed directly from the CLI:

```bash
erz pack-replay examples/program-packs/ingest-normalize --summary
erz pack-replay examples/program-packs/dedup-cluster --summary
erz pack-replay examples/program-packs/dedup-cluster --fixture new_ops_time_miss --summary
erz pack-replay examples/program-packs/dedup-cluster --include-fixture "new_*" --exclude-fixture "*security*" --summary
erz pack-replay examples/program-packs/dedup-cluster --fixture-class runtime_error --summary
erz pack-replay examples/program-packs/dedup-cluster --summary-file /tmp/dedup.summary.txt
erz pack-replay examples/program-packs/dedup-cluster --summary --json-file /tmp/dedup.summary.json
erz pack-replay examples/program-packs/dedup-cluster --fixture-class runtime_error --fixture-class-summary-file /tmp/dedup-runtime-error.summary.txt
erz pack-replay examples/program-packs/dedup-cluster --summary --strict-profile clean
erz pack-replay examples/program-packs/dedup-cluster --summary --strict-profile dedup-cluster-clean
erz pack-replay examples/program-packs/ingest-normalize --summary --strict-profile ingest-normalize-clean
erz pack-replay examples/program-packs/alert-routing --summary --strict-profile alert-routing-clean
erz pack-replay examples/program-packs/dedup-cluster --summary --strict-profile clean --expected-fixture-class-counts ok=4,expectation_mismatch=0,runtime_error=0
erz pack-replay examples/program-packs/dedup-cluster --summary --strict --expected-pack-id sprint-7-program-pack-2-dedup-cluster --expected-baseline-shape fixture-matrix --expected-mismatch-count 0 --expected-expectation-mismatch-count 0 --expected-runtime-error-count 0 --expected-rule-source-status ok
erz pack-replay examples/program-packs/alert-routing
erz pack-replay examples/program-packs/alert-routing --output /tmp/alert-routing-pack-replay.json
```

Current scope: both fixture-matrix baselines (`baseline.json` or `*.baseline.json` with `rules` + `fixtures`) and inline statement baselines that carry embedded `ev` samples. For inline baselines, `pack-replay` derives deterministic replay fixtures from the checked-in event statements plus the baseline rule set, so packs like `ingest-normalize/` no longer need a separate replay matrix just to reach the CLI lane. For CI gates, `--fixture <id>` can be repeated for exact cases, while `--include-fixture <glob>` and `--exclude-fixture <glob>` build deterministic slices in canonical pack order, include/exakt first, exclude second; `--fixture-class <ok|expectation_mismatch|runtime_error>` filters the replayed slice afterward by actual outcome class; `--summary` then reports `fixtures=<selected>/<total>` plus `fixture_classes=ok:<n>,expectation_mismatch:<n>,runtime_error:<n>`, while JSON consumers can read the same exclusive class labels per fixture under `fixtures[].fixture_class`, the aggregated counters from `summary.fixture_class_counts`, and the deterministic per-class fixture ids from `summary.fixture_class_ids`. `--summary-file <path>` now writes that same compact summary line for every pack run to a sidecar file, `--json-file <path>` writes the full JSON envelope side-by-side so summary stdout can hand off machine-readable results in the same run, `--fixture-class-summary-file <path>` remains available as the narrower compatibility path for fixture-class filtered runs, and `--output <path>` still mirrors the actual stdout lane. For the common checked-in green lane, `--strict-profile clean` is the generic short path: it auto-enables strict replay and gates `rule_source_status=ok` plus zero mismatches, zero expectation mismatches, and zero runtime errors without forcing brittle repeated fixture-id bundles into CI scripts. For the three checked-in packs, `--strict-profile ingest-normalize-clean`, `dedup-cluster-clean`, and `alert-routing-clean` go one step tighter by also pinning the expected `pack_id`, `baseline_shape`, total fixture count, and full `fixture_class_counts` histogram, so each pack can collapse to a single stable preset without extra count flags. For expected non-green baselines, `--strict` or `--strict-profile <preset>` plus `--expected-pack-id <pack-id>`, `--expected-baseline-shape <fixture-matrix|inline-statements>`, `--expected-fixture-count`, `--expected-total-fixture-count`, repeatable `--expected-selected-fixture <id>`, repeatable `--expected-ok-fixture <id>`, repeatable `--expected-expectation-mismatch-fixture <id>`, repeatable `--expected-runtime-error-fixture <id>`, `--expected-fixture-class-counts ok=<n>,expectation_mismatch=<n>,runtime_error=<n>`, `--expected-mismatch-count`, `--expected-expectation-mismatch-count`, `--expected-runtime-error-count` and `--expected-rule-source-status` turns the replay into an exact contract, keeping the raw replay outcome under `replay_status` while `status` becomes the gate result. `--expected-selected-fixture` checks the final deterministic `selected_fixture_ids` list after exact/glob/class filtering in canonical pack order, and the class-specific fixture selectors check the exact deterministic ids under `summary.fixture_class_ids.ok`, `summary.fixture_class_ids.expectation_mismatch` and `summary.fixture_class_ids.runtime_error`, so selector drift, green-partition drift, and failure-partition drift still trip strict replay even when the aggregate counters stay unchanged. If exact ids are overkill, `--expected-fixture-class-counts` can instead hard-gate the full deterministic histogram under `summary.fixture_class_counts`. `--expected-pack-id` gives custom packs the same explicit identity gate that the checked-in `*-clean` presets already use internally, without forcing another named preset, and `--expected-baseline-shape` does the same for fixture-matrix versus inline-statement baseline form. `--expected-mismatch-count` tracks the broad `summary.mismatch_count` including runtime errors, while `--expected-expectation-mismatch-count` isolates pure expectation drift.
